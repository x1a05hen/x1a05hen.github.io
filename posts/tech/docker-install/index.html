一、准备工作#

  1. 卸载旧版本

​ 旧版本的Docker以docker、docker.io或docker-engine为名。在尝试安装新版本之前，先卸载任何此类旧版本。

sudo apt-get remove docker docker-engine docker.io containerd runc

​ apt-get 显示没有包，则证明没有安装过。若以前安装过docker，则还需清除镜像、容器等：

sudo rm -rf /var/lib/docker
sudo rm -rf /var/lib/containerd
  1. apt换源

​ 备份源列表

sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak

​ 命令行打开sources.list文件

sudo vim /etc/apt/sources.list

​ 将源文件内容全部注释,并添加以下任意源

二、安装#

1. 设置仓库#

​ 在新主机上首次安装Docker引擎之前，需要设置Docker存储库。之后，可以从存储库安装和更新Docker。

  1. 更新 apt 包索引
sudo apt update
  1. 安装 apt 依赖包，用于通过 HTTPS 来获取仓库：
sudo apt install \
  apt-transport-https \
  ca-certificates \
  curl \
  gnupg-agent \
  software-properties-common
  1. 添加 Docker 的官方 GPG 密钥：
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

ok

  1. 9DC8 5822 9FC7 DD38 854A E2D8 8D81 803C 0EBF CD88 通过搜索指纹的后 8 个字符，验证您现在是否拥有带有指纹的密钥。
sudo apt-key fingerprint 0EBFCD88

pub   rsa4096 2017-02-22 [SCEA]
      9DC8 5822 9FC7 DD38 854A  E2D8 8D81 803C 0EBF CD88
uid           [ unknown] Docker Release (CE deb) <docker@docker.com>
sub   rsa4096 2017-02-22 [S]
  1. 使用以下指令设置官方稳定版仓库(使用阿里云镜像库)
sudo add-apt-repository \
   "deb [arch=amd64] https://mirrors.aliyun.com/docker-ce/linux/ubuntu \
   $(lsb_release -cs) \
   stable"

2.安装 Docker Engine#

​ 更新gpt

sudo apt update
  1. 安装最新版本的 Docker Engine 和 containerd
sudo apt install docker-ce docker-ce-cli containerd.io
  1. 安装特定版本的 Docker Engine:

​ 列出可用版本

apt-cache madison docker-ce

​ 使用第二列的版本号安装指定版本, 例如：5:20.10.23~3-0~ubuntu-focal

sudo apt install docker-ce=5:20.10.23~3-0~ubuntu-focal docker-ce-cli=5:20.10.23~3-0~ubuntu-focal containerd.io

​ 等待安装完成

3.测试#

sudo docker run hello-world

​ 遇到问题

Cannot connect to the Docker daemon at unix:///var/run/docker.sock. Is the docker daemon running?

​ 输入docker，会显示 –help，证明docker安装成功，是没有启动

cd /var/run
service docker start

​ 之后再测试，则没有问题

Hello from Docker!
This message shows that your installation appears to be working correctly.

三、其他工作#

添加用户#

sudo useradd -r -m -s /bin/bash x1a0
passwd x1a0

-s 指定命令解释器 bin/bash

指定主目录

usermod -d /home/test test

查看用户属性

cat /etc/passwd 

test❌1004:1005::/home/cat:/bin/bash 
用户名:口令:用户标识号:组标识号:注释性描述:用户主目录:命令解释程序

切换用户

su test

加入root

因为此文件只有r权限，在改动前需要增加w权限，改动后，再去掉w权限。

sudo chmod +w /etc/sudoers
sudo vim /etc/sudoers

# User privilege specification

root    ALL=(ALL:ALL) ALL

x1a0   ALL=(ALL:ALL) ALL

# Members of the admin group may gain root privileges

%admin ALL=(ALL) ALL


# Allow members of group sudo to execute any command

%sudo   ALL=(ALL:ALL) ALL


sudo chmod -w /etc/sudoers

(ctrl+d)退出用户

sudo userdel x1a0 shanchu

如果要删除用户，请按下面操作进行，分为3步：

1、执行userdel命令：sudo userdel zhaoligang

2、删除用户目录命令：sudo rm -rf /home/zhaoligang

3、删除用户权限相关配置：删除或者注释掉/etc/sudoers中关于要删除用户的配置，否则无法再次创建同名用户。

添加ssh秘钥登录#

ssh-keygen
cd ~/.ssh
cat id_rsa.pub >> authorized_keys

如此便完成了公钥的安装。 此外，为了确保连接成功，输入如下指令以保证以下文件权限正确：

chmod 600 authorized_keys
chmod 700 ~/.ssh

service sshd restart

配置非root用户使用docker#

​ 以非 root 用户运行 Docker，需要将用户添加到 docker 组

sudo usermod -aG docker $USER

​ 重启 Docker

sudo service docker restart

​ 退出当前用户重新登录，再查看

docker -v

​ ok

配置Docker镜像加速#

​ 阿里云容器，生成自己的加速地址

​ 登录：https://cr.console.aliyun.com/

​ 点击 “创建我的容器镜像”，在左侧镜像工具下拉栏，找到镜像加速器，即可看到自己专属加速地址。

​ 可以通过修改daemon配置文件/etc/docker/daemon.json来使用加速器

sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://qe6cwxjg.mirror.aliyuncs.com"]
}
EOF
# 重启docker
sudo service docker restart
### 注意
WSL没有systemctl ，所以只能用service
